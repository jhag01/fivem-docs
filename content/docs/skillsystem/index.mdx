skillSystem is a flexible, framework-agnostic skill and progression system for FiveM. Built with performance in mind — a single server thread handles all decay, a single save tick handles all players, and all skill data is cached server-side with no per-player threads.

<ResourceLinks repo="skillSystem" />

## Features

- Fully dynamic skill definitions — add new skills with zero code changes
- XP-based progression with a configurable exponential curve per skill
- Per-skill decay with four modes: online, offline, both, or none
- Maximum decay cap per skill — skills decay to a floor, not to zero
- Offline decay calculated on login based on time away
- Framework auto-detection — supports Qbox and ESX out of the box
- Clean NUI with a quick menu pill and a full side panel
- Temporary XP boosts and debuffs from any script — cooldown-gated per boost, stackable, config presets or fully ad-hoc definitions
- Internal, ox_lib, or custom notification support
- In-game admin menu to view and modify any player's skills, with ace- and framework-group-based permission checks
- Configurable keybind with tap (quick menu) and hold (full panel) behaviour
- Debug logging toggle for development
